Shader Anatomy!

A unity shader is split up into two main sections: "shaderlab" (configuration) part , and "CG" (hlsl) part. Make a new shader and paste this in, to play around with.

If you are familiar with html/css/js, you can kind of think ShaderLab as HTML, HLSL as Javascript.

Shader "LearnSurface/Color"
{
    Properties
    {
        _Color ("Color", Color) = (1,1,1,1)
        _MainTex ("Albedo (RGB)", 2D) = "white" {}
        _Glossiness ("Smoothness", Range(0, 1)) = 0.5
        _Metallic ("Metallic", Range(0, 1)) = 0.0
    }
    SubShader
    {
        Tags { "RenderType"="Opaque" }
		
        ZWrite On
        Cull Back
        Blend Off
		
        CGPROGRAM	
        #pragma surface surf Standard fullforwardshadows vertex:vert addshadow dithercrossfade
        #pragma target 5.0
        
        struct Input
        {
            float2 texcoord;
        };
		
        float4 _Color;			
        sampler2D _MainTex; 	
        float _Glossiness;  		
        float _Metallic;    	
        
        void vert(inout appdata_full app, out Input input)
        {
            UNITY_INITIALIZE_OUTPUT(Input, input);
            input.texcoord = app.texcoord.xy;
        }

        void surf(Input input, inout SurfaceOutputStandard o)
        {
            o.Albedo = tex2D(_MainTex, input.texcoord).rgb * _Color.rgb;
            o.Smoothness = _Glossiness;
            o.Metallic = _Metallic;
            o.Normal = float3(0, 0, 1);
        }	
        ENDCG
    }
    FallBack "Diffuse"
}

Unity HLSL stuff Reference

Input Struct

Appdata, or appdata_full, is the struct containing per-vertex data, such as UVs, vertex color, normal, etc.

struct appdata_full { float4 vertex : POSITION; float4 tangent : TANGENT; float3 normal : NORMAL; float4 color : COLOR; float4 texcoord : TEXCOORD0; float4 texcoord1 : TEXCOORD1; float4 texcoord2 : TEXCOORD2; float4 texcoord3 : TEXCOORD3; float4 texcoord4 : TEXCOORD4; float4 texcoord5 : TEXCOORD5; };

Surface Output Struct

Surface Output is the struct you create and return from the 'surf' function, it contains parameters like color, normal, metallness, for unity to use in its lighting.

struct SurfaceOutputStandard { float3 Albedo; // Base Color float3 Normal; // Tangent space normal float3 Emission; // Emission color float Metallic; // 1 = metallic 0 = non-metallic float Smoothness; // 0 = rough 1 = smooth float Occlusion; // Ambient Occlusion float Alpha; // Opacity value, 1 = opaque, 0 = transparent };

Unity Shaderlab flags Reference

ZWrite

The ZWRite flag controls to depth, IE if it should obscure things that are behind it or not!
ZWrite command in ShaderLab reference


// Hide objects it it's in front ZWrite On // Let other objects, even ones behind, draw on top of us, based on draw queue order. ZWrite Off

ZTest

Ztest, Whether a pixel should be discared based on the "current" depth value at that pixel.
There are more than I listed here but these are the useful ones.
ZTest command in ShaderLab reference


// Always draw the obejct, ignoring if it's behind something or not. ZTest Always // Default value, draws the object if it's *closer* than what's alreayd there ZTest Less // Draws the object only if it's *behind* something. (good for stuff like outlines to see things through walls) ZTest GEqual

Blend

Blend parameter describes how the object should blend with the backrgound when it's drawn. (Additive, etc)
The docs for this is split across two pages for some reason
ShaderLab: Blending
Blend command in ShaderLab reference

// Traditional transparency Blend SrcAlpha OneMinusSrcAlpha // Premultiplied transparency Blend One OneMinusSrcAlpha // Additive Blend One One // Soft Additive Blend OneMinusDstColor One // Multiplicative Blend DstColor Zero // 2x Multiplicative Blend DstColor SrcColor

Cull

Cull controls Back-face culling / Front face culling. IE whether to draw the front face or back face.
Cull command in ShaderLab reference

// Standard backface culling, hide back faces Cull Back // Draw the object two-sided! Cull Off // Hide front faces Cull Front
